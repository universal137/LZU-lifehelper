# 兰大生活助手 - 系统技术文档

## 一、系统简介

### 1.1 系统概述

"兰大生活助手"是一款面向兰州大学师生的校园一站式桌面服务平台，基于 PySide6（Qt6）框架开发，采用 SQLite 作为本地数据库，实现了二手交易、场馆预约、校园出行、活动管理和生活圈等核心功能。

### 1.2 功能模块

系统包含以下功能模块：

| 模块 | 功能说明 |
|------|----------|
| **用户认证** | 登录/注册、角色区分（学生/教师/管理员）、密码修改、个人资料编辑 |
| **二手市场** | 商品发布/浏览/搜索、分类筛选、价格排序、商品收藏、留言互动 |
| **场馆预约** | 场馆时段查看、在线预约/取消、日期筛选、余量可视化 |
| **校园出行** | 校车班次查询、座位可视化、一键购票、发车倒计时 |
| **活动中心** | 活动发布/报名、报名进度可视化、名单导出（CSV） |
| **生活圈** | 动态发布/浏览、点赞/评论、分类筛选、实时搜索 |
| **管理后台** | 用户管理（封禁/解封）、商品审核、预约管理、操作日志、数据可视化仪表盘 |

### 1.3 技术路线

```
┌─────────────────────────────────────────────────┐
│                   表现层 (Views)                  │
│  PySide6/Qt6 + QSS样式 + 自定义动画组件           │
├─────────────────────────────────────────────────┤
│                   业务层 (Models)                 │
│  AppModel 类：业务逻辑 + 数据访问 + 会话管理       │
├─────────────────────────────────────────────────┤
│                   数据层 (Database)               │
│  SQLite3 + WAL模式 + 增量迁移 + 索引优化           │
└─────────────────────────────────────────────────┘
```

**核心组件：**
- `desktop_app.py` — 应用入口，异常处理与日志初始化
- `desktop/models.py` — 数据库模型、业务逻辑、ORM（约1526行）
- `desktop/views.py` — UI界面、交互逻辑、自定义组件（约3038行）
- `desktop/style.qss` / `style_dark.qss` — 亮色/暗色主题样式表
- `assets/images/` — 商品分类占位图、登录背景图等静态资源

## 二、编码规范

### 2.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| **类名** | 大驼峰（PascalCase） | `AppModel`、`MainWindow`、`BarChart` |
| **函数/方法** | 小驼峰（camelCase） | `refresh_home()`、`toggle_like()` |
| **私有方法** | 前缀下划线 | `_build_ui()`、`_create_tables()` |
| **常量** | 全大写下划线 | `SCHEMA_VERSION`、`ASSETS_DIR` |
| **变量** | 小写下划线 | `slot_rows`、`admin_user_filter` |
| **信号** | 小驼峰 | `login_success`、`notify` |
| **QSS对象名** | 小驼峰+.setObjectName | `"pageTitle"`、`"kpiValue"`、`"navList"` |

### 2.2 代码风格

- **缩进**：4个空格，不使用Tab
- **行宽**：建议不超过120字符
- **引号**：统一使用双引号（`"`）
- **类型注解**：使用 `from __future__ import annotations` 启用延迟求值
- **导入顺序**：标准库 → 第三方库 → 项目内部模块，每组之间空一行

### 2.3 注释规范

- **模块级**：文件顶部简要说明用途
- **类级**：用 docstring 说明类的职责（如 `FormDialog`、`BarChart`）
- **方法级**：复杂逻辑添加行内注释，简单方法不强制注释
- **TODO**：用 `# TODO:` 标记待完成项

### 2.4 QSS规范

- 选择器使用 objectName（`#pageTitle`）而非类型选择器
- 颜色值使用十六进制（`#C9A962`）或 rgba（`rgba(201, 169, 98, 0.3)`）
- 渐变使用 `qlineargradient()` 函数
- 不使用 Qt 不支持的属性（`box-shadow`、`letter-spacing`、`line-height` 等）

### 2.5 Git规范

- **提交信息**：使用中文描述，格式为 `类型: 简要说明`
- **分支策略**：main（主分支）、dev（开发分支）、feature/*（功能分支）
- **类型**：feat（新功能）、fix（修复）、refactor（重构）、style（样式）

## 三、数据库设计

### 3.1 数据库概述

采用 SQLite3 作为本地嵌入式数据库，启用 WAL（Write-Ahead Logging）模式提升并发读性能。数据库文件位于 `data/lzu_lifehelper.db`。

**关键特性：**
- 外键约束：`PRAGMA foreign_keys = ON`
- WAL模式：`PRAGMA journal_mode = WAL`
- 版本管理：通过 `PRAGMA user_version` 实现增量迁移
- 索引优化：11个索引覆盖高频查询字段

### 3.2 数据表结构

系统共包含 **14张表**：

#### 用户表 `users`
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,          -- 登录账号
    display_name TEXT NOT NULL,             -- 显示姓名
    password_hash TEXT NOT NULL,            -- PBKDF2加盐哈希
    role TEXT NOT NULL CHECK(role IN ('student', 'teacher', 'admin')),
    status TEXT NOT NULL DEFAULT 'active' CHECK(status IN ('active', 'banned')),
    college TEXT NOT NULL,                  -- 学院/单位
    bio TEXT NOT NULL,                      -- 个人简介
    avatar_color TEXT NOT NULL,             -- 头像背景色
    created_at TEXT NOT NULL
);
```

#### 商品表 `products`
```sql
CREATE TABLE products (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    category TEXT NOT NULL,                 -- 书籍/运动/数码/日用/资料/器材
    campus TEXT NOT NULL,                   -- 榆中校区/城关校区
    price REAL NOT NULL,
    description TEXT NOT NULL,
    image_path TEXT,                        -- 商品图片路径
    seller_id INTEGER NOT NULL,
    status TEXT NOT NULL DEFAULT 'normal' CHECK(status IN ('normal', 'removed')),
    created_at TEXT NOT NULL,
    FOREIGN KEY (seller_id) REFERENCES users(id)
);
```

#### 商品留言表 `product_messages`
```sql
CREATE TABLE product_messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    product_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    content TEXT NOT NULL,
    created_at TEXT NOT NULL,
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 场馆表 `venues`
```sql
CREATE TABLE venues (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,                     -- 场馆名称
    category TEXT NOT NULL,                 -- 室内运动/球类运动/活动空间
    campus TEXT NOT NULL,
    location TEXT NOT NULL
);
```

#### 场馆时段表 `venue_slots`
```sql
CREATE TABLE venue_slots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    venue_id INTEGER NOT NULL,
    slot_date TEXT NOT NULL,                -- 日期 YYYY-MM-DD
    slot_time TEXT NOT NULL,                -- 时段 如 "08:00-10:00"
    capacity INTEGER NOT NULL DEFAULT 1,
    UNIQUE (venue_id, slot_date, slot_time),
    FOREIGN KEY (venue_id) REFERENCES venues(id) ON DELETE CASCADE
);
```

#### 预约表 `bookings`
```sql
CREATE TABLE bookings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    slot_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    status TEXT NOT NULL DEFAULT 'active' CHECK(status IN ('active', 'cancelled', 'admin_cancelled')),
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    FOREIGN KEY (slot_id) REFERENCES venue_slots(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 校车线路表 `shuttle_routes`
```sql
CREATE TABLE shuttle_routes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    route_name TEXT NOT NULL,
    from_campus TEXT NOT NULL,
    to_campus TEXT NOT NULL,
    station TEXT NOT NULL,                  -- 乘车点
    departure_time TEXT NOT NULL,           -- 发车时间 HH:MM
    base_capacity INTEGER NOT NULL,         -- 总座位数
    status TEXT NOT NULL DEFAULT 'normal' CHECK(status IN ('normal', 'disabled'))
);
```

#### 校车车票表 `shuttle_tickets`
```sql
CREATE TABLE shuttle_tickets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    route_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    ride_date TEXT NOT NULL,                -- 乘车日期
    status TEXT NOT NULL DEFAULT 'active' CHECK(status IN ('active', 'cancelled')),
    created_at TEXT NOT NULL,
    FOREIGN KEY (route_id) REFERENCES shuttle_routes(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 活动表 `activities`
```sql
CREATE TABLE activities (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    category TEXT NOT NULL,                 -- 公益/体育/学术/文艺/成长
    organizer_id INTEGER NOT NULL,
    location TEXT NOT NULL,
    start_time TEXT NOT NULL,
    capacity INTEGER NOT NULL,
    summary TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'normal' CHECK(status IN ('normal', 'cancelled')),
    created_at TEXT NOT NULL,
    FOREIGN KEY (organizer_id) REFERENCES users(id)

```

#### 活动报名表 `activity_registrations`
```sql
CREATE TABLE activity_registrations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    user_id INTEGER NOT NULL,

    created_at TEXT NOT NULL,
    UNIQUE (activity_id, user_id),

    FOREIGN KEY (user_id) REFERENCES users(id)

```

#### 动态表 `moments`
```sql
CREATE TABLE moments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    category TEXT NOT NULL,                 -- 校园通知/失物招领/吐槽问答/活动/二手交易
    content TEXT NOT NULL,
    image_path TEXT,
    status TEXT NOT NULL DEFAULT 'normal' CHECK(status IN ('normal', 'removed')),
    created_at TEXT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id)

```

#### 动态点赞表 `moment_likes`
```sql
CREATE TABLE moment_likes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    moment_id INTEGER NOT NULL,

    UNIQUE (moment_id, user_id),
    FOREIGN KEY (moment_id) REFERENCES moments(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id)

```

#### 动态评论表 `moment_comments`
```sql
CREATE TABLE moment_comments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    moment_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    content TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'normal' CHECK(status IN ('normal', 'removed')),
    created_at TEXT NOT NULL,
    FOREIGN KEY (moment_id) REFERENCES moments(id) ON DELETE CASCADE,
```

#### 管理日志表 `admin_logs`
```sql
CREATE TABLE admin_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    admin_id INTEGER NOT NULL,
    action TEXT NOT NULL,                   -- 操作类型
    target_type TEXT NOT NULL,              -- 操作对象类型
```

#### 收藏表 `favorites`
```sql
CREATE TABLE favorites (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
```

### 3.3 索引设计

```sql
-- 商品查询优化
CREATE INDEX idx_products_status_category ON products(status, category);
CREATE INDEX idx_products_seller ON products(seller_id);

-- 预约查询优化
CREATE INDEX idx_bookings_user_status ON bookings(user_id, status);
CREATE INDEX idx_bookings_slot ON bookings(slot_id, status);

-- 校车票查询优化
CREATE INDEX idx_shuttle_tickets_route_date ON shuttle_tickets(route_id, ride_date);
CREATE INDEX idx_shuttle_tickets_user ON shuttle_tickets(user_id, status);

-- 活动查询优化
CREATE INDEX idx_activities_status ON activities(status);
CREATE INDEX idx_activity_registrations_user ON activity_registrations(user_id, status);

-- 动态查询优化
CREATE INDEX idx_moments_category_status ON moments(category, status);
CREATE INDEX idx_moment_comments_moment ON moment_comments(moment_id);
CREATE INDEX idx_moment_likes_moment ON moment_likes(moment_id);
```

### 3.4 数据库迁移机制

系统采用基于版本号的增量迁移机制，通过 `PRAGMA user_version` 管理版本：

```python
SCHEMA_VERSION = 4

MIGRATIONS = {
    3: ["CREATE INDEX ...", ...],   # 版本3：添加索引
    4: ["CREATE TABLE favorites ..."],  # 版本4：添加收藏表
}
```

- 全新安装：直接创建所有表和索引
- 已有数据库：按版本号依次执行增量迁移 SQL
- 数据安全：迁移仅执行 `CREATE TABLE IF NOT EXISTS` 和 `CREATE INDEX IF NOT EXISTS`，不会删除或修改现有数据

## 四、关键实现说明

### 4.1 安全机制

- **密码存储**：PBKDF2-SHA256 加盐哈希（10万次迭代），兼容旧版 SHA-256 自动迁移
- **XSS防护**：所有 HTML 内容通过 `html.escape()` 转义用户输入
- **输入校验**：商品标题≤50字符、描述≤2000字符、价格>0、活动时间格式校验
- **权限控制**：学生/教师/管理员三级角色，管理操作需二次确认

### 4.2 自定义UI组件

| 组件 | 说明 |
|------|------|
| `BarChart` | 渐变色柱状图，支持参考线和数据标签 |
| `LineChart` | 折线图，渐变填充区域+数据点 |
| `PieChart` | 饼图，10色配色+图例+百分比 |
| `Toast` | 滑入/淡出通知，支持队列和动画 |
| `AnimatedStackedWidget` | 页面切换淡入动画 |
| `FormDialog` | 通用表单对话框，自动生成表单+字数统计 |
| `LoginVisualPanel` | 登录页背景面板，paintEvent绘制校园图+遮罩 |

### 4.3 性能优化

- **图标缓存**：场馆/校车/moment分类图标缓存到类变量，避免重复创建QPixmap
- **防抖搜索**：商品/动态/日志搜索使用QTimer 300ms防抖
- **WAL模式**：SQLite启用WAL，提升并发读性能
- **processEvents**：刷新时插入事件处理，保持UI响应

### 4.4 主题系统

支持亮色/暗色一键切换：
- `style.qss` — 亮色主题（深蓝+金色配色）
- `style_dark.qss` — 暗色主题（Catppuccin风格）
- 通过 `toggle_theme()` 方法动态切换QSS文件
